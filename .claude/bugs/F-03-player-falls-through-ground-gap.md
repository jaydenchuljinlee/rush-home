# 플레이어가 지면 타일 사이 공백으로 낙하

## 증상
게임 시작 직후 지면이 스크롤되면서 플레이어(X=0)가 타일 사이 13유닛 공백 구간에 위치하게 되어 Y=-5261까지 무한 낙하함.

## 에러 로그
에러 없음. 단순 물리 낙하.

## 오브젝트 상태
- **Player localPosition**: (x=-1.92, y=-5261.25, z=0) — 지면 기준점(y=0)에서 5261 아래
- **GroundTile_0**: center X=-11.82, 커버 범위 X[-19.82, -3.82]
- **GroundTile_1**: center X=17.43, 커버 범위 X[9.43, 25.43]
- **GAP**: X=-3.82 ~ X=9.43 (약 13유닛) — 플레이어(X=-1.92)가 이 공백에 위치

## 스크린샷
플레이 시작 12초 시점 캡처. 타이머 00:11.02 확인됨.

## 판정 기준 위반
- 기대: Player Y 위치 >= 지면 높이, NullReferenceException 없음
- 실제: Player Y = -5261.25 (지면 아래로 무한 낙하)

## 추정 원인
GroundScroller.tileWidth = 20 이지만 실제 타일 스프라이트 크기는 16x2.
씬 초기 배치 시 타일 간격이 tileWidth 기준(20)이 아닌 다른 값으로 설정되어 있어,
Tile_0(X=0)과 Tile_1(X≈29)사이에 ~13유닛 공백이 발생함.
게임 시작 후 tiles가 왼쪽으로 스크롤되면 플레이어가 이 공백에 떨어짐.

## 수정 방향
1. GroundScroller.tileWidth를 실제 스프라이트 너비(16)에 맞게 수정 (tileWidth=16)
2. 또는 씬에서 초기 타일 위치를 0, 16, 32 (tileWidth=16 기준)로 재배치
3. 또는 초기 타일 배치를 tileWidth=20 기준 0, 20, 40으로 재배치하되 스프라이트 크기를 20으로 변경
