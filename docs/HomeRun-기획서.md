# HomeRun - 프로젝트 기획서

> 퇴근길을 배경으로 한 2D 사이드뷰 러너 게임

## 📌 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | HomeRun (코드네임) |
| **장르** | 2D 사이드뷰 엔드리스/타임어택 러너 |
| **플랫폼** | 모바일 (iOS / Android) |

| **서버/백엔드** | PlayFab (랭킹/인증) |

## 🎮 게임 컨셉

### 한 줄 설명
> "퇴근 후 집에 도착하기까지, 이 도시의 온갖 장애물을 뚫고 가장 빠르게 귀가하라."

### 핵심 루프
1. 플레이어가 회사에서 출발
2. 자동으로 오른쪽으로 달림 (자동 러닝)
3. 점프 / 슬라이딩 / 회피로 장애물 극복
4. 집에 도착할 때까지의 시간 측정
5. 글로벌 랭킹에 등록
6. 실패 시 재도전

### 타겟 유저
- 20~40대 직장인
- 짧은 시간(1~3분) 안에 즐길 수 있는 캐주얼 게임 선호
- 랭킹 경쟁에 동기부여받는 유저

## 🗺️ 레벨 디자인 컨셉

### 맵 구성 (테마별 청크)
퇴근길의 내러티브를 유지하기 위해 청크를 테마별로 분류합니다.

| 구간 | 테마 | 장애물 예시 |
|------|------|------------|
| 1 | 회사 앞 | 바쁜 직장인, 회전문 |
| 2 | 횡단보도 | 신호등, 지나가는 차 |
| 3 | 지하철역 | 계단, 개찰구, 혼잡한 인파 |
| 4 | 환승 구간 | 좁은 통로, 에스컬레이터 |
| 5 | 버스 정류장 | 줄 서 있는 사람, 달려오는 버스 |
| 6 | 골목길 | 웅덩이, 자전거, 킥보드 |
| 7 | 집 앞 | 계단, 택배 상자 |

### 난이도 곡선
```
0~30초:   쉬움  - 장애물 1~2개, 간격 넓음
30~60초:  보통  - 장애물 3~4개, 복합 패턴 시작
60~120초: 어려움 - 연속 장애물, 타이밍 정밀 요구
120초~:   지옥  - 최대 속도, 패턴 예측 불가
```

### 장애물 유형
- **점프 장애물**: 웅덩이, 계단, 앉은 사람
- **슬라이딩 장애물**: 낮은 간판, 가로수 가지
- **회피 장애물**: 좌우로 피해야 하는 킥보드, 문
- **타이밍 장애물**: 여닫히는 문, 지나가는 버스

## 🛡️ 치팅 방지 설계

랭킹이 핵심인 게임이라 치팅 방지는 필수입니다.

### 클라이언트 측
- 완주 시간은 서버 타임스탬프 기반으로만 기록
- 체크포인트 통과 시점을 서버에 전송
- 민감한 값은 암호화/난독화

### 서버 측 (PlayFab CloudScript)
- 세션 시작/종료 시각을 서버에서 기록
- 비현실적 완주 시간 자동 reject (예: 10초 이내)
- 체크포인트 간 속도 검증
- 일일 리더보드로 의심 기록의 장기 영향 최소화
